# Dataset Documentation ‚Äî FraudLens AI

## Source

**Kaggle: Credit Card Fraud Detection**
- URL: https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud
- File: `creditcard.csv` (~144 MB)
- Transactions: 284,807 rows over two days (September 2013, European cardholders)

## Setup

Download `creditcard.csv` from Kaggle and place it at:
```
ml/data/raw/creditcard.csv
```

> [!IMPORTANT]
> The `ml/data/raw/` directory is gitignored. **Never commit the raw dataset.**

---

## Schema

The dataset has **31 columns** (no header renaming required):

| Column | Type | Description |
|--------|------|-------------|
| `Time` | float64 | Seconds elapsed since the first transaction in the dataset |
| `V1`‚Äì`V28` | float64 | Principal Components from PCA (anonymised features) |
| `Amount` | float64 | Transaction amount in USD |
| `Class` | int64 | Target label: `0` = Normal, `1` = Fraud |

### Class Distribution

| Class | Count | Percentage |
|-------|-------|------------|
| Normal (0) | 284,315 | 99.827% |
| Fraud (1) | 492 | 0.173% |

> **Imbalance Ratio**: ~578:1 (Normal:Fraud). SMOTE is applied to the training split to address this.

## Preprocessing Workflow

The pipeline is orchestrated by `PreprocessingPipeline` in `ml/src/preprocessing/pipeline.py`.

### Step 1 ‚Äî Load (`dataset_loader.py`)

Reads `creditcard.csv` with full file validation:
- Checks file existence, `.csv` extension, readability, and parsability.
- Returns a typed `pd.DataFrame`.

### Step 2 ‚Äî Validate (`validator.py`)

Runs 9 automated checks:
- Schema: all 31 columns present
- Datatypes: all numeric
- Missing values: null percentage per column
- Duplicates: row duplicate rate
- Target integrity: only 0 and 1 values
- Target distribution: fraud rate logging
- Unique values: constant column detection
- Memory usage profiling

Saves `ml/reports/data_validation_report.json`.

### Step 3 ‚Äî EDA (`eda.py`)

Generates 8 headless (non-interactive) plots:

| Filename | Description |
|----------|-------------|
| `class_imbalance.png` | Bar + pie chart of class distribution |
| `amount_distribution.png` | Amount histogram by class |
| `time_distribution.png` | Time density by class |
| `fraud_vs_amount.png` | Boxplot and log-scale comparison |
| `correlation_heatmap.png` | Lower-triangle feature correlation |
| `pca_scatter.png` | 2D PCA scatter coloured by class |
| `feature_statistics.png` | Heatmap of normalised feature stats |
| `outlier_summary.png` | IQR-based outlier counts per feature |

Saves `ml/reports/eda_report.md` and all PNGs to `ml/reports/figures/`.

### Step 4 ‚Äî Preprocess (`preprocessor.py`)

**Scaling strategy:**
- `Amount` and `Time` ‚Üí **StandardScaler** (zero mean, unit variance)
- `V1`‚Äì`V28` ‚Üí **left untouched** (already PCA-transformed by the dataset provider)

The fitted scaler is saved to `ml/artifacts/scalers/amount_time_scaler.pkl` for inference reuse without re-fitting.

### Step 5 ‚Äî Split (`splitter.py`)

Stratified train/validation/test split preserving the class ratio in each subset:

| Split | Ratio | ~Rows |
|-------|-------|-------|
| Train | 70% | ~199,365 |
| Validation | 15% | ~42,721 |
| Test | 15% | ~42,721 |

Stratification ensures that the 0.173% fraud rate is maintained identically in each split.

### Step 6 ‚Äî SMOTE (`smote.py`)

**Synthetic Minority Over-sampling Technique** is applied **exclusively to the training split**.

> [!CAUTION]
> SMOTE must **never** be applied to validation or test data. The `SMOTEResampler` class enforces this with an explicit `SMOTEError` if misused.

**Why SMOTE on training only?**
- Prevents data leakage into evaluation sets
- Keeps validation and test metrics representative of real-world conditions
- Generates synthetic fraud samples via k-nearest-neighbour interpolation

| Parameter | Value |
|-----------|-------|
| `sampling_strategy` | `"minority"` |
| `k_neighbors` | `5` |
| `random_state` | `42` |

### Step 7 ‚Äî Save Arrays

Six NumPy arrays saved to `ml/data/processed/`:
- `X_train.npy`, `X_val.npy`, `X_test.npy` ‚Äî Feature matrices
- `y_train.npy`, `y_val.npy`, `y_test.npy` ‚Äî Target vectors

### Step 8 ‚Äî Metadata (`metadata.py`)

Saves `ml/reports/metadata.json` capturing full pipeline provenance:
- Dataset version and timestamp
- Row counts per split (before and after SMOTE)
- Class distributions at every stage
- Scaler type and fitted parameters
- SMOTE configuration
- Split ratios and random seed

## Reproducibility

All stochastic operations use `random_state=42`:
- `train_test_split` (stratified)
- `SMOTE.fit_resample`
- `PCA` in EDA scatter plot

This seed is defined in `ml/config/config.py` as `RANDOM_SEED = 42` and propagated automatically.
